# MSA Builder

**Memory · Soul · Agent —— 建立你与 AI 之间的底层协作契约。支持分层架构。**

MS&A 是一个"自我认知协作系统"，由三个文件组成：

- **Memory（记忆）**——你的事实层快照：你是谁、在什么处境里、做过什么
- **Soul（灵魂）**——你的思维操作系统：你为什么会这样思考、底层模式是什么
- **Agent（协作原则）**——你与 AI 之间的契约：你希望 AI 如何与你协作

AI 在每次对话前读这三个文件，不需要你重新交代背景、反复纠正同样的问题。

## MSA 2.0

进阶用户可将系统升级为分层架构：

- **入口层** — 各平台入口文件（CLAUDE.md、AGENTS.md），只做适配
- **核心常驻层** — Memory、Soul、Agent，每次对话加载
- **按需细则层** — references/ 子文件，只在特定任务时加载

核心理念：**各司其职，零冗余，薄适配，一键齐。**
文件边界由执行频率决定：每次必须执行的规则 → 入口文件；不是每次都必须执行的偏好 → Agent.md。

## 适用场景

- 你频繁使用 AI 协作，但每次对话都要重新交代背景
- 你发现 AI 总是写"很对"但"不像你"
- 你想建立一套可复用的自我认知框架，让 AI 真正理解你的思维方式和协作偏好

## 使用方法

### 方式一：通过 Claude Code 触发

在 Claude Code 对话中输入：

| 你说 | 执行 |
|:----|:-----|
| `搭建 MSA` / `建立自我认知系统` | 完整搭建流程，交付三份文件 |
| `更新 Memory/Soul/Agent` | 克制地更新指定文件 |
| `诊断 MSA` | 检查现有三文件的质量 |
| `对齐 MSA` / `同步 MSA` | 系统性检查三文件同步状态 |

### 方式二：自行阅读 SKILL.md

`SKILL.md` 是完整的流程文档，包含了采集信息的方法、三文件起草标准和交付规范。你可以直接阅读并按流程手动搭建。

## 输出示例

搭建完成后你会得到三份 Markdown 文件：

```
--- Memory.md ---
（你的事实层快照）

--- Soul.md ---
（你的思维模式提炼）

--- Agent.md ---
（你和 AI 的协作契约）
```

## 设计原则

- **克制优于追加**——能删的先删，能合并的合并。这条信息没有 AI 会不会犯错？不会就不写
- **面向 AI 写**——AI 需要精确、无歧义、可执行的规则，不是抒情散文
- **Soul 更新门槛最高**——除非你自己重新定义了某个底层认知，否则不轻易改灵魂
- **改一条不动三条**——信息有变只动那条，不连带重写整个文件
