# Align 模式

## 目标

把当前会话产生的稳定变化同步到唯一真实源，同时删除重复、过期和失效内容。

## 流程

1. 记录对齐前 Git 状态和已有修改。
2. 通读当前会话及受影响文件，只提取跨会话仍有价值的信息。
3. 按架构分流到入口、MSA、references、ALIGNMENT、Skills 或外部资料源。
4. 优先改写和删除，不把对齐理解为追加日志。
5. 检查 Gotchas 是否应进入、保留、沉淀为规则或退出。
6. 运行 `scripts/verify_system.py <instance-root>`。
7. 对照对齐前状态，确认没有覆盖未获准修改。

## Gotchas 生命周期

Gotcha 必须包含：发生了什么、已确认的原因、以后如何做。

- 只记录真实发生、影响严重或重复出现的问题。
- 活跃 Gotchas 最多 3 条。
- 严重事故可以暂时与稳定规则并存，但必须明确退出条件。
- 根因被结构性消除或提醒不再有价值时删除；历史由 Git 保存。

## 提交

只有用户明确授权提交或精确触发已约定包含提交的 `[对齐]` 时才创建本地 commit。

- 只暂存本轮获准文件，不使用无边界的 `git add .`。
- 提交前检查 staged diff。
- 不自动 push、打 tag 或发布 release。
