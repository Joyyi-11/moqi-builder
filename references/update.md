# Update 模式

## 目标

更新已确认的信息，同时保持单一真实源和文件边界。

## 流程

1. 从磁盘读取相关文件和 Git 状态。
2. 明确变化是事实、稳定模式、协作关系、运行规则、领域细则还是临时信息。
3. 判断应修改旧条目、删除旧条目还是新增；优先改写，不只追加。
4. 说明文件清单；结构性变化先确认。
5. 修改后检查所有引用、路由和适配器。
6. 运行实例校验器。

## 写入门槛

- **Memory**：信息已确认，会改变当前协作判断，并且不应由外部资料源承担。
- **Soul**：模式在多个真实场景中反复出现，有证据支撑。
- **Agent**：关系偏好稳定，描述的是如何共同工作。
- **AGENTS**：不执行就会直接导致行为错误、安全风险或高频摩擦。
- **reference**：只在清晰的任务类型中需要，而且入口有加载路由。
- **Skill**：完整流程可复用，有独立触发边界。

未经确认、可能过期、只是本次过程或已经存在权威来源的信息不写入。
